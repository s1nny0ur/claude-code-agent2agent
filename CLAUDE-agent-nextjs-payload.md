## TypeScript types come from the Payload agent

All Payload content types live in `{{DIR_A}}/src/payload-types.ts` — generated by the
Payload agent via `pnpm payload generate:types`. Do not edit it manually.

Before using a new type:

```bash
# Confirm typegen is done
cat {{BRIDGE}}/typegen-status   # must be "done"

# Confirm the type exists
grep "HeroBlock" {{DIR_A}}/src/payload-types.ts
```

Import types from the Payload agent's worktree:

```typescript
import type { HeroBlock, Page } from '../../payload-types'
// or use relative path to {{DIR_A}}/src/payload-types.ts
```

Always read the schema contract before writing fetch code:

```bash
cat {{BRIDGE}}/schema-contract.md
```

## Fetching Payload data

Payload exposes a REST API at `/api/[collection]`. Use the `getPayload()` helper
for server-side fetching in this project.

Before writing a fetch function, study existing patterns:

```bash
# Existing fetch utilities
ls {{DIR_B}}/src/utilities/
cat {{DIR_B}}/src/utilities/getDocument.ts
cat {{DIR_B}}/src/utilities/getGlobals.ts

# Check how pages are fetched in existing routes
cat {{DIR_B}}/src/app/(frontend)/[slug]/page.tsx
```

REST API shape (check schema-contract.md for specifics):

```bash
# Example: fetch a page by slug with full block depth
GET /api/pages?where[slug][equals]=home&depth=2

# Example: fetch a collection list
GET /api/posts?limit=10&depth=1
```

## Block component workflow

When notified of a new block (check `block-registry-queue.md`):

1. Check the queue:
   ```bash
   cat {{BRIDGE}}/block-registry-queue.md
   ```

2. Wait for typegen:
   ```bash
   cat {{BRIDGE}}/typegen-status   # must be "done"
   ```

3. Read the type shape:
   ```bash
   cat {{BRIDGE}}/schema-contract.md
   grep "HeroBlock" {{DIR_A}}/src/payload-types.ts -A 30
   ```

4. Study existing block components for patterns:
   ```bash
   ls {{DIR_B}}/src/blocks/
   cat {{DIR_B}}/src/blocks/Banner/Component.tsx
   cat {{DIR_B}}/src/blocks/Content/Component.tsx
   ```

5. Create `{{DIR_B}}/src/blocks/Hero/Component.tsx`

6. Register in `{{DIR_B}}/src/blocks/RenderBlocks.tsx`:
   ```bash
   cat {{DIR_B}}/src/blocks/RenderBlocks.tsx
   # Add the new case/import following the existing pattern
   ```

7. Confirm back:
   ```bash
   {{BRIDGE}}/send-to-agent.sh a "Hero Component.tsx created and registered in RenderBlocks.tsx"
   ```

## How to answer questions

When the Payload agent asks about your codebase:

1. **Read the actual source.** Open the real files — don't answer from memory.
2. **Trace the full rendering path.** Understand how data flows from fetch → component.
3. **Be precise.** Use exact prop names and types. Missing a prop = runtime error.
4. **Cite your sources.** Reference file and line number so they can verify.
5. **Flag gotchas.** Missing `depth` param, image URL construction, missing `?` on optional fields.
